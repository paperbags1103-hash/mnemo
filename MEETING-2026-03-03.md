한 줄 소개: 자료 수집·정리·자동화·편집이 가능한 웹앱 (lorien 기반 지식 관리 시스템)

- lorien = 백엔드 (지식 그래프 추론 엔진)
- mnemo = 프론트엔드 (웹 UI 인터페이스)

## 피어그룹 리서치 결과 (researcher)

- "AI 에이전트용 Obsidian 웹앱" 컨셉은 시장에 없음 → 독점 포지션
- 메모리 레이어(Mem0 등)는 있지만 편집 가능한 웹 UI + 자동 수집 파이프라인 + 그래프 뷰 조합 없음
- 경쟁사: Mem0 ($249/월, API만), Obsidian (인간용), NotebookLM (읽기 전용)

### 재사용 가능 컴포넌트
- **TipTap** — React 마크다운 에디터 오픈소스 (Notion 동급 품질)
- **vis.js** — 그래프 뷰 (lorien serve에 이미 있음 ✅)
- **Firecrawl** — 웹 수집 자동화 오픈소스
- **shadcn/ui** — Obsidian 스타일 다크 UI

## 결정사항

### MVP 범위
- **첫 기능: 마크다운 에디터**
- 그래프 뷰: v1.1로 미룸
- 수집 자동화: v1.1 이후

### 기술 스택
- **프론트엔드: Vite + React** (Opus 어드바이저 + 팀 만장일치)
  - signal 프로젝트 동일 스택 → 즉시 시작 가능
  - TipTap React 생태계 가장 성숙
- **에디터: TipTap**
- **UI: shadcn/ui**

### 아키텍처
```
mnemo
├── notes.db (SQLite 사이드카)   ← 원문 저장 (id, title, content, created_at, updated_at)
└── lorien REST API (port 7331)  ← 지식 추출 저장 (Facts, Entities, Rules)
```

흐름:
1. 에디터에서 노트 작성 → notes.db 저장
2. 저장 시 lorien ingest_text() 호출 → 자동 지식 추출
3. source_ref에 note id 기록 → lorien ↔ 원문 연결 추적

원문 관리: lorien 스키마 별도 수정 없이 SQLite 사이드카로 분리
